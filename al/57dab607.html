1.样式4的移植记录(python处理图片部分)

  • 下载python：2021年如何通过官网下载和安装Python和PyCharm最新版本
    python版本：3.9.9 64-bit
    我没下pycharm
    顺便可以改安装路径：python安装路径可以更改吗
  • 不会在GitHub仓库中新建文件夹：GitHub仓库中如何创建文件夹
  • 不会克隆仓库：如何远程克隆仓库
  • 没安装pillow库：python安装pil图像库
  • 全选代码，shift+enter运行（vscode）

报错1：source directory not exist!

解决方法：把路径改成绝对路径

报错2：not enough values to unpack (expected 2, got 1)

解决方法：上面路径的文件夹里不能有子文件夹，因为子文件夹没有.xx的后缀，用split()函数分类（分隔符为.）会卡住。最好只放图片。

报错3：failed to push some refs to github

解决方法：

  1. 关于main和master的问题
  2. 克隆仓库的命令：git clone https://github.com/alnas1/photos.git
    如果是ssh拉取：git clone git@github.com:Alnas1/photos.git
  3. 把source下的photos文件夹删掉，在source文件夹中右键打开命令行，执行克隆仓库的命令。
  4. 进入克隆下来的仓库，随便放张图进去测试。
  5. 执行下面三条命令看能不能成功。
  6. 成功了就能在GitHub仓库看到变更。
  7. 把tool.py文件里的 os.system('git push origin master')改为 os.system('git push origin main')
  8. 再次运行文件tool.py
  9. 然后我发现不能把相册文件夹放在博客目录下，不然相册提交会提交一次图，博客部署又会提交一次图。

2.样式4的移植记录(hexo配置部分)

  1. data.json保存在photos下。”F:\1-blog\MyWeb\source\photos\data.json”
  2. 注意核对index.md中链接css等文件的路径。我放在”F:\1-blog\MyWeb\themes\next\source\ _data"
  3. 完全跟着教程建立文件夹即可，不要自找麻烦。
  4. ins.js文件中链接的修改。这是正确的图片链接格式（注意main前面那个raw）：https://github.com/Alnas1/photos/raw/main/photos/2021-12-15_suibian%20(2).jpg
    关于这点请看：本地图片上传到GitHub，MarkDown使用Github图片地址
